Loading Data Using {here} package

Hello there,

This tutorial is designed to help you, my fellow {r} beginner, to hopefuly have an easy start with {r}. I remember my first attempt to use {r} in Multiple Regression class last year. The very first step was to load the data and I spent hours!!! (not kidding) just to successfully doing it. My professor gave us his code with a note “change this direction to your own path!” and arrrghh it almost made me give up! Until I learn about {here} package! The following is the objective of {here} package:

The goal of the here package is to enable easy file referencing in project-oriented workflows. In contrast to using {setwd()}, which is fragile and dependent on the way you organize your files, here uses the top-level directory of a project to easily build paths to files.

In more simplified words, the function {here ()} customizes the path on your local computer. Therefore, everyone can use the same line of command to load data on each of their computer. This particularly very useful When you collaborate and share your code with your collage. No more error warning.

Here are three simple steps:

1. Installation

Open your r project and Install {here} package and load it on your computer

install.packages("here")
library (here)

2. Check your path.

To check you can run this line of code on your console. It should give back the path of your current project

here()

3. Load data from your folder:

  • create a name of your data using the assignment operator {<-}
  • use a reader function (for base r: read.csv). )

For this example, I want to load student.csv that is located inside my data folder.

df <- read.csv(here("folder_name", "file_name"))

df <- read.csv(here ("data", "student.csv"))

You can also use import() from {rio}

df <- import(here ("data", "student.csv"))

or read_csv from {readr}

df <- read_csv(here ("data", "student.csv"))

If someone else use this code, as long as they have the same file on the data folder inside their local R Project, this code will run just fine!

That’s it!

Let us all thank Jenny Bryan the creator of {here package} for her genius work! You can also check her github page to learn more other functions in here package.
